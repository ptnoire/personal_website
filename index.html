February 3rd, 2023

Update DOM Algorithm

I've learned a lot over the past couple months when it comes to DOM manipulation. This was the first time I'd ever used a few of these built in features and done something that cuts down the memory usage of the website when updating elements on the page.


Long story short (which I want to go in-depth but won't here) createRange() .createContextualFragment() is creating a virtual DOM so that I can compare the nodes to the current DOM, then using a conditional statement we will change the text content only if the node has a value of textcontent [nodeValue], then just below I'm specifically updating a dataset attribute used to modify the page's values from another feature. Do you think this is cool? I think in a large scale application it maybe a bit much but for this specific project is was really fancy.

                    
    update(data) {
        this.data = data;
        const newMarkup = this._generateMarkup(data);
        const newDOM = document.createRange().createContextualFragment(newMarkup);
        const newElements = Array.from(newDOM.querySelectorAll('*'));
        const curElements = Array.from(this._parentElement.querySelectorAll('*'));
        newElements.forEach((newEl, i) => {
            const curEl = curElements[i];
            if(!newEl.isEqualNode(curEl) && newEl.firstChild?.nodeValue.trim() !== '') {
                curEl.textContent = newEl.textContent;
            };
            if(!newEl.isEqualNode(curEl)) {
                Array.from(newEl.attributes).forEach(attr => curEl.setAttribute(attr.name, attr.value));
            }
        })
    }
